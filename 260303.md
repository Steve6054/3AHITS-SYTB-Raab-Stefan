# GitHub

Wir verwenden ein GitHub Repository und VS Code um unsere Berichte zu schreiben.

Im GitHub müssen wir zuerst ein Repository erstellen und diesen dann in ein lokales Kali-Verzeichnis klonen:

```bash
git clone <Link zum GitHub>
```

Danach findet man das GitHub-Repository in dem Verzeichnis in dem wir den Befehl ausgeführt haben.

# VS Code

Als erstes müssen wir den Browser öffnen und nach `VS Code install on Linux` suchen. Hier das erste Ergebnis öffnen und `.debpackage(64-bit)` installieren und danach folgenden Befehl im Downloads-Verzeichnis ausführen:

```bash
sudo apt install ./code_1.109.5-1771531656_amd64.deb
```

Jetzt können wir VS Code in einem beliebigen Verzeichnis starten, indem wir folgenden Befehl im gewünschten Verzeichnis ausführen:

```bash
code .
```

# Berichte schreiben

Um Berichte dann auch im GitHub zu haben, öffnen wir VS Code in dem bereits geklonten GitHub-Verzeichnis.

Jetzt kann man hier die Berichte schreiben und wenn man fertig ist die Änderungen `commiten`, also an die Remote-GitHub Datenbank übergeben. Das macht man indem man ganz links auf den 3.Menüpunkt klickt und hier `commit` ausführt. Dabei muss eine Nachricht angegeben werden (also was geändert wurde). Nach dem commiten müssen wir auch noch auf `Sync Changes` klicken. Das liegt daran, dass beim `commit` selbst die Änderungen erst noch lokal gespeichert werden. Beim `Sync` wird dann `Push` und `Pull` ausgeführt, also alle lokalen Änderungen auf die Remote-Datenbank übergeben und alle Änderungen die sich bereits dort befinden (z.Bsp bei Gruppenarbeiten) auf das lokale System übernommen.

Beim ersten mal commiten müssen wir auch noch den usernamen und die email festlegen:

```bash
┌──(kali㉿kali)-[~/SYTB/Berichte]
└─$ git config --global user.name "Stefan Raab"
                                                                                                                                                                  
┌──(kali㉿kali)-[~/SYTB/Berichte]
└─$ git config --global user.email stefan.raab@htl-braunau.at
```

Zusätzlich wäre es noch zu empfehlen unter "File" die Option "Auto Save" zu aktivieren, weil man vor jedem commit sowieso das File speichern muss und es so automatisch geht.
